Objective 1

Follow a software development process to analyze a problem, and to design, build, and test software solutions.

Project Runner GitLab

Runner used a Scrum software development methodology. The team met frequently and much of the development process consisted of peer programming. When the team's objectives or goals for the project changed, the planning was then capable of easily being adjusted accordingly.

Tag! You're It! GitHub

Tag was developed using a sprint objective system to track which portions of the project were completed and what was scheduled next. Due to the team being a size of one, production was incredibly flexible. Setbacks (bugs, defects, etc.) were always finished within the same sprint they were discovered.

Objective 2

Develop software development skills using more than one programming language and development environment.

Asobouyo! Gist

Asobouyo! (or Let's Play in English) utilizes the Java programming language on the Android platform. A MySQL database was used to store user information on a remote server, requiring familiarity with both SQL and the Linux operation system. Furthermore, PHP requests to the database were called by Java HTTP requests.

Appointment Manager GitHub

Programming basics were applied by implementing a custom String, DateTime, and Array class in C++ for use in a simple Appointment Manager. Visual Studio was used in order to familiarize oneself with feature-heavy and industry standard IDEs.

Asteroids GitHub

Asteroids was written in Python using the PyGame libraries. This game is cross-platform and can run on any machine with the Python and PyGame libraries installed. Due to Python being a loosely-typed language, it can be written in simple text editors with little to no syntax highlighting with ease and was therefore written in the Python IDLE.

Objective 3

Design and implement software solutions across multiple platforms.

Reline.github.io GitHub

This website was built using the Bootstrap framework and the jQuery JavaScript libraries. It is built to be dynamic and accustomed across all platforms and browsers. Shown here is a script written in jQuery to retrieve the GitHub feed of a user.

Bob & Jay Breakout GitHub

Bob&Jay was developed in PyGame and runs everywhere.

Intermediary GitHub

The concept of Intermediary came from the need to know the pronunciation of certain Chinese characters on the go, and lead to implementing support for all available locales on the Android platform (if the language is installed on the device).

Objective 4

Design and implement software solutions for mobile devices, embedded systems, or standalone systems.

Earthquake Hazards GitHub

Earthquakes was developed for the Android mobile platform. It was a simple project to help become familiar with the Android development environment by using the many features offered by the Android operating system, including (but not limited to) a SQLite database, User Preference Activities, List Adapters, and Alarms.

NESPi GitHub

The NESPi was designed for the Raspberry Pi 2 using an Arduino Leonardo board for the input interface.

Objective 5

Document the planning and development of software applications using industry standard tools, techniques, and processes.

Java Function Tester GitHub

The documentation for this project included high-level description of general and specific requirements, including scope, user interface, maintainability, and adaptability (e.g. using this software with newer versions of Java).

Tag! You're It! GitHub

Tag used sprints to document stages of the project. Further documentation shows wireframes, design plans, and prior planning. The project is also version controlled on GitHub for the purpose of tracking progress and safe storage.

Objective 6

Within software solutions describe, implement, and analyze data structure techniques such as lists, trees, hash tables, graphs, along with sorting and searching algorithms.

Frederick's Quest for Facial Hair GitHub

The entire map that Frederick travels throughout this game is generated from a monospaced textfile which is parsed for key symbols with the appearance of a map. Each symbol represents a particular tile type within the game. The background and characters are then drawn and blitted to the screen appropriately.

Perspective Rectifier GitHub

A custom matrix class was designed to work more smoothly with the calculus used within the project by using a system of jagged arrays. Matrix multiplication and other operations were also implemented.
