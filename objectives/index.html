Objective 1

Follow a software development process to analyze a problem, and to design, build, and test software solutions.

Project Runner GitLab

Runner used a Scrum software development methodology. The team met often and more often than not did programming with one another or in close vicinity of each other. When our desired outcome of the project changed, the planning was adjusted accordingly.

Tag! You're It! GitHub

Tag was developed using a sprint objective system by tracking which portions of the project were completed and what was scheduled next. Due to the team being a size of one, production was incredibly flexible, so any setbacks (bugs, defects, etc.) were always finished within the same sprint they were discovered.

Objective 2

Develop software development skills using more than one programming language and development environment.

Asobouyo! Gist

Asobouyo! (or Let's Play in English) utilizes the Java programming language on the Android platform. A MySQL database was used to store user information on a remote server, requiring familiarity with both SQL and the Linux operation system. Furthermore, PHP requests to the database were called by Java HTTP requests.

Appointment Manager GitHub

Programming basics were learned by implementing a custom String, DateTime, and Array class in C++. Visual Studio was used in order to familiarize oneself with feature-heavy IDEs and specifically aimed for the Windows OS.

Asteroids GitHub

Asteroids was written in Python using the PyGame libraries. This game is cross-platform and can run on any machine with the Python and PyGame libraries installed. Due to Python being a loosely-typed language, it can be written in simple text editors with little to no syntax highlighting with ease. Not being completely familiar with IDEs yet, this project was written in Python IDLE.

Objective 3

Design and implement software solutions across multiple platforms.

Reline.github.io GitHub

This website was built using the Bootstrap libraries and the jQuery JavaScript libraries. It is built to be dynamic and accustomed across all platforms and browsers.

Bob & Jay Breakout GitHub

As Bob&Jay was developed in PyGame, it can be executed across any platform that can use the Python compiler and PyGame libraries.

Intermediary GitHub

Intermediary's original purpose was to pronounce Kanji whose furigana was unknown to the user. Quick access was needed, so the applicaton was developed for the Android platform. It eventually lead to implementing support for all available locales on the Android platform, as long as the language was installed.

Objective 4

Design and implement software solutions for mobile devices, embedded systems, or standalone systems.

Earthquake Hazards GitHub

Earthquakes was developed for the Android mobile platform. It was a simple project to help become familiar with the Android development environment by using a little bit of everything. These include a SQLite database, User Preference Activities, List Adapters, and Alarms.

NESPi GitHub

The NESPi was implemented using a Raspberry Pi as the CPU and Arduino Leonardo board as the input interface.

Objective 5

Document the planning and development of software applications using industry standard tools, techniques, and processes.

Java Function Tester GitHub

Documentation included high-level description of general and specific requirements, including scope, UI, and adaptability (e.g. using this software with new versions of Java).

Tag! You're It! GitHub

Tag used sprints to document stages of the project. Further documentation shows wireframes, design plans, and prior planning. The project is also version controlled on GitHub for the purpose of tracking progress and safe storage.

Objective 6

Within software solutions describe, implement, and analyze data structure techniques such as lists, trees, hash tables, graphs, along with sorting and searching algorithms.

Frederick's Quest for Facial Hair GitHub

The entire map that Frederick travels throughout this game was created by a monospaced textfile which is then parsed for certain symbols as if it were a map. Each symbol represents a particular tile type within the gameat the start. The background and characters are then drawn and blitted to the screen appropriately.

Perspective Rectifier GitHub

A custom matrix class was designed to work more smoothly with the calculus side of the project using arrays. Matrix multiplication, addition, and such was implemented for utility.
